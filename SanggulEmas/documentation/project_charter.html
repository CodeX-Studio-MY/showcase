2. Business Context

Sanggul Emas Sdn Bhd is an established Oil & Gas technical solutions provider in Malaysia, expanding its portfolio into digital monitoring solutions. To support this market entry, Sanggul Emas requires a demonstrable, white-label IoT Asset Monitoring Platform suitable for upsale of existing maintenance solutions to O&G end-customers.

CodeX Studio is engaged as the technical delivery partner, providing cloud platform, software development, and lifecycle support. Hardware is procured and installed by Sanggul Emas separately.

Strategic note: This POC functions both as a working platform and a sales-enablement asset. Success is measured not only by technical delivery, but also by Sanggul Emas's ability to use the platform credibly in front of prospective clients within the demo window.

3. Project Objectives

Primary

  1. Deliver an end-to-end IoT monitoring platform covering tank-level and pump-vibration use cases.
  2. Validate the technical architecture (LoRaWAN → LORIOT → Azure → Web App) under realistic conditions.
  3. Equip Sanggul Emas with a demo-ready Tank Monitoring module within 1 month of kickoff for early prospect engagement.
  4. Establish a reusable, white-labellable foundation that can scale into production engagements.

Secondary

  1. Document a clear migration path to production-grade compliance (self-hosted ChirpStack, IEC 62443, SOC 2 readiness).
  2. Produce sales-enablement collateral (architecture diagrams, security statement, compliance roadmap) for use in Sanggul Emas's prospect pitches.

4. Scope

✓ In Scope

  • Core Platform: Azure Function ingestion, Table Storage, .NET REST API, React dashboard
  • Microsoft Entra ID SSO/MFA
  • Tank Level Monitoring module — ingestion, visualization, threshold alerts
  • Pump Vibration Monitoring module — v-RMS per ISO 10816, alerts
  • LORIOT (community LNS) webhook integration
  • Single-tenant deployment with Sanggul Emas branding
  • Dev + staging environments
  • UAT and 1-Year Cloud & Care Plan
  • Sensor data simulator for demo and testing

✗ Out of Scope (POC)

  • Hardware procurement, installation, commissioning
  • FFT vibration analysis (separate quotation if revived)
  • Modbus M-mode integration (future phase)
  • Self-hosted ChirpStack migration (roadmap only)
  • Multi-tenant architecture
  • Mobile native applications
  • Optional add-ons not in PO: multi-language, audit log, PDF export, email digest, downlink commands, data export, alert history
  • End-customer site connectivity, SIM provisioning

6. Success Criteria

Mid-POC Milestone — Tank Monitoring Demo (~15 June 2026)

  1. Tank-level data flowing end-to-end (sensor or simulator → dashboard) with under 60-second latency.
  2. Dashboard branded with Sanggul Emas identity, demo-ready on a desktop browser.
  3. At least one threshold-based alert configured and triggering correctly.
  4. Microsoft Entra ID SSO/MFA functional for at least 2 test users.
  5. Sanggul Emas's sales team can independently log in and walk a prospect through the dashboard without CodeX Studio present.

Final UAT (~13 July 2026)

  1. Both modules (tank + vibration) operational with live or simulated data.
  2. v-RMS visualization conformant to ISO 10816 zones.
  3. Alert engine functional for both modules with configurable thresholds.
  4. UAT executed and signed off by Sanggul Emas's appointed reviewer.
  5. All deliverables (D1–D11) issued; compliance & migration roadmap accepted as sales-enablement asset.

7. Assumptions

  1. Hardware (Aircom end nodes, Aircom gateway, Monitran vibration sensors, tank-level sensors) is supplied, installed, configured and commissioned by Sanggul Emas before Sprint 3. (Not necesarily commissioned as long as gateway is online and receiving some data for LORIOT integration in Sprint 2).
  2. Sensor count: 4 tank-level + 2 pump-vibration sensors (pending formal confirmation at kickoff).
  3. Aircom gateway operates in F-mode (cloud) for the POC.
  4. Sanggul Emas provides branding assets (logo, colour palette, typography) by end of Week 1.
  5. Sanggul Emas appoints a single decision-making point-of-contact for project governance.
  6. Azure tenant arrangement (Sanggul Emas, CodeX-managed interim, or end-customer) confirmed at kickoff or end of Week 1.
  7. Internet/SIM connectivity for the LoRaWAN gateway is provisioned by Sanggul Emas.
  8. UAT will be conducted within Week 8, with consolidated feedback returned within 3 working days.

8. Constraints

  1. Effort capped at 250 man-hours; overruns trigger a written Change Request at RM 175–200/hour per Quotation #000017.
  2. Timeline fixed at 8 weeks from kickoff (18 May 2026 → 13 July 2026).
  3. LORIOT community LNS used for POC; production-grade key custody requires post-POC migration.
  4. LoRaWAN payload size (~51 bytes) constrains telemetry richness; FFT/raw waveform is not feasible over LoRaWAN.
